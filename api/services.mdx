title: Services
description: Core service classes that handle business logic in ShelfWise
---

ShelfWise uses a service layer architecture to keep controllers thin and encapsulate business logic. All services are located in `app/Services/` and handle complex operations, data validation, and coordination between models.

## Core Principles

- **Thin Controllers**: All business logic resides in services
- **Transaction Management**: Critical operations use database transactions
- **Cache Invalidation**: Services handle cache management automatically
- **Tenant Isolation**: All operations are tenant-scoped for security
- **Audit Logging**: Services log all significant operations

## ProductService

Manages product creation, updates, variants, and packaging types.

**Location**: `app/Services/ProductService.php`

### create()

Creates a new product with variants and packaging types.

<ParamField path="data" type="array" required>
  Product data including name, type, variants, and packaging
  
  <Expandable title="data properties">
    <ParamField path="name" type="string" required>
      Product name
    </ParamField>
    <ParamField path="product_type_slug" type="string" required>
      Product type slug (e.g., 'general', 'pharmacy', 'retail')
    </ParamField>
    <ParamField path="category_id" type="int">
      Product category ID
    </ParamField>
    <ParamField path="description" type="string">
      Product description
    </ParamField>
    <ParamField path="has_variants" type="bool" default="false">
      Whether product has multiple variants
    </ParamField>
    <ParamField path="is_active" type="bool" default="true">
      Product active status
    </ParamField>
    <ParamField path="custom_attributes" type="array">
      Type-specific attributes
    </ParamField>
    <ParamField path="variants" type="array">
      Array of variant data (if has_variants is true)
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="tenant" type="Tenant" required>
  The tenant instance
</ParamField>

<ParamField path="shop" type="Shop" required>
  The shop instance
</ParamField>

<ResponseField name="return" type="Product">
  Created product with relationships loaded
</ResponseField>

```php
use App\Services\ProductService;

$productService = app(ProductService::class);

$product = $productService->create([
    'name' => 'Pain Relief Tablets',
    'product_type_slug' => 'pharmacy',
    'category_id' => 5,
    'description' => '500mg paracetamol tablets',
    'has_variants' => false,
    'sku' => 'MED-001',
    'price' => 15.99,
    'cost_price' => 8.50,
    'base_unit_name' => 'Pack',
    'custom_attributes' => [
        'requires_prescription' => false,
        'drug_class' => 'analgesic',
    ],
], $tenant, $shop);
```

## OrderService

Handles order lifecycle management including creation, status transitions, and fulfillment.

**Location**: `app/Services/OrderService.php`

### createOrder()

Creates a new order with items.

<ParamField path="tenant" type="Tenant" required>
  Tenant instance
</ParamField>

<ParamField path="shop" type="Shop" required>
  Shop instance
</ParamField>

<ParamField path="items" type="array" required>
  Array of order items
  
  <Expandable title="item properties">

    </ParamField>
    <ParamField path="sellable_id" type="int" required>

    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="createdBy" type="User" required>
  User creating the order
</ParamField>

<ParamField path="customer" type="User">
  Customer user (optional)
</ParamField>

<ResponseField name="return" type="Order">
  Created order with items and totals calculated
</ResponseField>

```php

$orderService = app(OrderService::class);

            'sellable_id' => 123,
            'quantity' => 2,
            'unit_price' => 50.00,
        ],
    ],
    createdBy: auth()->user(),
    customer: $customer,
    customerNotes: 'Handle with care',
    shippingCost: 10.00
);
```

### confirmOrder()

Confirms an order and reserves inventory.

<ParamField path="order" type="Order" required>
  Order to confirm
</ParamField>

<ParamField path="user" type="User" required>
  User confirming the order
</ParamField>

<ResponseField name="return" type="Order">
  Confirmed order with reserved inventory
</ResponseField>

<Accordion title="Stock Reservation Logic">
  When an order is confirmed:
  1. Uses pessimistic locking to prevent race conditions
  2. Checks available stock (quantity - reserved_quantity)
  3. Atomically increments reserved_quantity
  4. Throws exception if insufficient stock
  5. Updates order status to CONFIRMED
</Accordion>

```php
$order = $orderService->confirmOrder($order, auth()->user());
```

### fulfillOrder()

Fulfills a confirmed order by deducting inventory and creating stock movements.

<ParamField path="order" type="Order" required>
  Confirmed order to fulfill
</ParamField>

<ParamField path="user" type="User" required>
  User fulfilling the order
</ParamField>

<ResponseField name="return" type="Order">
  Fulfilled order with status updated to PROCESSING
</ResponseField>

```php
$order = $orderService->fulfillOrder($order, auth()->user());
```

### cancelOrder()

Cancels an order and releases reserved inventory.

<ParamField path="order" type="Order" required>
  Order to cancel
</ParamField>

<ParamField path="user" type="User" required>
  User canceling the order
</ParamField>

<ParamField path="reason" type="string">
  Cancellation reason
</ParamField>

<ResponseField name="return" type="Order">
  Cancelled order
</ResponseField>

```php
$order = $orderService->cancelOrder(
    $order, 
    auth()->user(), 
    'Customer requested cancellation'
);
```

## POSService

Handles Point-of-Sale operations including quick sales and session management.

**Location**: `app/Services/POSService.php`

### createQuickSale()

Creates an immediate sale with payment and stock deduction.

<ParamField path="shop" type="Shop" required>
  Shop where sale is occurring
</ParamField>

<ParamField path="items" type="array" required>
  Array of sale items
</ParamField>

<ParamField path="customerId" type="int">
  Customer ID (optional)
</ParamField>

<ParamField path="paymentMethod" type="string" default="cash">
  Payment method (cash, card, mobile_money)
</ParamField>

<ParamField path="amountTendered" type="float" default="0">
  Cash amount tendered
</ParamField>

<ParamField path="options" type="array">
  Additional options (discount_amount, notes, etc.)
</ParamField>

<ResponseField name="return" type="Order">
  Completed order with DELIVERED status
</ResponseField>

<Accordion title="POS Sale Process">
  1. Validates stock availability using pessimistic locking
  2. Creates order with DELIVERED status
  3. Calculates totals including tax and discounts
  4. Creates order items
  5. Records stock movements
  6. Creates payment record
  7. Returns completed order
</Accordion>

```php
use App\Services\POSService;

$posService = app(POSService::class);

$order = $posService->createQuickSale(
    shop: $shop,
    items: [
        [
            'variant_id' => 123,
            'quantity' => 2,
            'unit_price' => 50.00,
            'packaging_type_id' => 1,
        ],
    ],
    customerId: 45,
    paymentMethod: 'cash',
    amountTendered: 120.00,
    options: [
        'discount_amount' => 10.00,
        'notes' => 'Loyalty discount applied',
    ]
);
```

**Location**: `app/Services/CartService.php`

Retrieves or creates a cart for the current session or customer.

<ParamField path="shop" type="Shop" required>

</ParamField>

## Service Best Practices

<Accordion title="Transaction Management">
  All services use database transactions for data consistency:
  
  ```php
  return DB::transaction(function () use ($data) {
      // Multiple database operations
      $model = Model::create($data);
      $related = $model->related()->create($relatedData);
      
      // All succeed or all fail
      return $model;
  });
  ```
</Accordion>

<Accordion title="Cache Invalidation">
  Services automatically invalidate relevant caches:
  
  ```php
  Cache::tags([
      "tenant:{$tenantId}:products:list",
      "tenant:{$tenantId}:product:{$productId}",
  ])->flush();
  ```
</Accordion>

<Accordion title="Tenant Isolation">
  All queries are scoped to the current tenant:
  
  ```php
  Product::query()
      ->where('tenant_id', $tenant->id)
      ->where('shop_id', $shop->id)
      ->get();
  ```
</Accordion>

<Accordion title="Error Handling">
  Services throw exceptions with clear messages:
  
  ```php
  if (!$order->canEdit()) {
      throw new Exception(
          'Order cannot be edited in current status: ' . 
          $order->status->value
      );
  }
  ```
</Accordion>
